Disambiguation¶

In the context of the Elsevier Fingerprint Engine, ‘Disambiguation’ means: establishing the correct meaning for a term candidate. After the process (activity) of term finding, the annotation storage may contain a number of term annotations. Those term annotations point to concepts. At this stage we can execute a number of steps to rule out or affirm specific word meanings. As context is the only clue we have at this point to establish the correct meaning for a word, all disambiguation methods have the same schema:

  • the context is inspected
  • the context is compared against a context model for the target term
  • an annotation is added to the target term to annotate its correctness

Disambiguation is performed by the activity annotate terms. This activity must point to a disambiguator static object.

There are the following types of disambiguators:

  • Term Annotator
  • Wide Context Disambiguator
  • Alignment Disambiguator

Disambiguator Static Objects¶

Wide Context Disambiguator¶

<!-- disambiguation by wide context -->
<dis:WideContextDisambiguator xaml:Key="WideContextNAL" LowerRanking="ignore" HighestRanking="confirmed"  >
    <dis:WideContextDisambiguator.DataSource>
        <cc:DatabaseDataSource ConnectionString="Server=(local);Database=NAL__;Trusted_Connection=Yes;"
                    Query="SELECT FromID AS context, ToID AS target, Value as distance, [Key] AS sign
                                FROM ItemRelations WHERE ([Key]='RuleOut' OR [Key]='Require')" />
    </dis:WideContextDisambiguator.DataSource>
</dis:WideContextDisambiguator>
